# Phishing Email Investigation

## Overview

This project demonstrates the investigation and analysis of a phishing email impersonating Microsoft. The objective was to identify phishing indicators, assess potential risks, and document findings following a SOC Analyst investigation process.

## Project Objectives

* Analyze a suspicious email
* Identify phishing indicators
* Detect social engineering techniques
* Assess the threat level
* Document findings and recommendations

## Investigation Summary

The email claimed to be a Microsoft account security alert and urged the recipient to recover their account immediately. During the investigation, multiple phishing indicators were identified, including domain impersonation, urgency-based messaging, and a potential credential harvesting attempt.

## Phishing Indicators Identified

* Suspicious sender domain
* Microsoft brand impersonation
* Urgency and fear tactics
* Credential harvesting attempt
* Social engineering techniques

## Threat Assessment

| Category    | Value               |

| Threat Type | Credential Phishing |
| Severity    | High                |

## Recommendations

* Verify sender domains before taking action.
* Avoid clicking suspicious links or buttons.
* Report phishing emails to the security team.
* Enable Multi-Factor Authentication (MFA).
* Conduct regular security awareness training.

## Skills Demonstrated

* Email Security Analysis
* Phishing Detection
* Threat Assessment
* Security Documentation
* SOC Investigation Workflow
* Cybersecurity Reporting

## Conclusion

The investigation confirmed that the email was a phishing attempt designed to trick users into revealing their Microsoft account credentials. The email utilized domain impersonation, social engineering, and urgency-based tactics commonly observed in phishing campaigns.
